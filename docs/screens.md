# 必要画面リスト — ゲーム神社（game-jinja）

DGX Spark ハッカソン 2026-06-13 / プロトタイプ方針：**とにかく「ゲームとして動くもの」を見せる**こと優先。
操作は全画面共通で **↑↓←→ ＋ Z ＋ C のみ**（テキスト入力なし・C1）。

## 画面一覧

| # | 画面 | 状態キー | 目的 | 主な操作 | 遷移先 | 優先度 |
|---|------|---------|------|---------|--------|--------|
| 1 | **タイトル** | `title` | 「ゲーム神社」表示・開始 | `Z`＝はじめる | → 2択 | **P0** |
| 2 | **メイン（プレイ）** | `play` | 授かったゲームを iframe で全画面プレイ | ↑↓←→ / Z / C（ゲーム本体が処理） | → 結果 | **P0** |
| 3 | 2択質問 | `quiz` | 3 問の 2 択で「欲しいゲームの気配」を決める | ←→ 選択・`Z` 決定 | → 祈り | P1 |
| 4 | 祈り | `pray` | 神社モチーフ＋祈り演出 | `Z` 連打/長押しでゲージ | → 生成待ち | P1 |

## プロトタイプ最短ライン（P0 だけで成立させる）

`title → play → result` の 3 画面が回れば「ゲームとして動く」デモになる。
- まず **タイトルで Z → ゲームが始まり → 遊べて → 結果が出てタイトルに戻る** を通す。
- 2択/祈り/生成待ち（P1）は最短ラインが通った後に挿す（演出・体験の肉付け）。
- プレイ画面のゲーム本体は、サーバ生成（/api/pray）が間に合わない場合に備え、フロント内蔵のデモゲーム 1 本にフォールバックして必ず動く状態にする。

## 各画面の受け入れ条件（抜粋）

- タイトル：`Z` で次へ。マウス・テキスト入力なし。
- プレイ：iframe にゲームを読み込み、↑↓←→/Z/C で操作でき、**3 分以内に終了**して結果へ（C3）。
- 結果：必ず到達し、`Z` 再挑戦 / `C` タイトルで次の来場者へ回せる。
